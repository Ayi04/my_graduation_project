# 图书馆座位预约系统实现文档

## 1. 系统架构

### 1.1 整体架构
- **架构风格**: 前后端分离架构
- **后端**: Spring Boot + Spring Data JPA + MySQL
- **前端**: Vue.js + Vite
- **部署方式**: 前后端独立部署

### 1.2 核心组件
- **后端核心模块**:
  - 控制器层 (Controller): 处理HTTP请求和响应
  - 服务层 (Service): 实现业务逻辑
  - 数据访问层 (Repository): 与数据库交互
  - 实体层 (Entity): 定义数据模型

- **前端核心模块**:
  - 路由管理 (Router): 处理页面导航
  - 状态管理 (Store): 管理全局状态
  - 组件 (Components): 可复用的UI元素
  - 视图 (Views): 页面级组件

## 2. 技术栈

### 2.1 后端技术
- **框架**: Spring Boot 2.x
- **ORM**: Spring Data JPA
- **数据库**: MySQL 8.0
- **构建工具**: Maven
- **认证**: 基于用户名密码的简单认证
- **CORS**: 跨域资源共享配置

### 2.2 前端技术
- **框架**: Vue 3
- **构建工具**: Vite
- **路由**: Vue Router
- **状态管理**: Vuex
- **UI组件**: 自定义组件

## 3. 系统功能模块

### 3.1 用户管理模块
- **用户注册**: 支持普通用户、工作人员和管理员注册
- **用户登录**: 基于用户名和密码的认证
- **个人信息管理**: 更新个人信息、修改密码、上传头像
- **信用分管理**: 记录用户信用分，支持禁言功能
- **违规记录管理**: 记录和查询用户违规行为

### 3.2 座位管理模块
- **区域管理**: 新增、编辑、删除区域
- **座位管理**: 新增、编辑、删除座位
- **座位状态管理**: 实时更新座位状态（可用、已预约、已占用）

### 3.3 预约管理模块
- **座位预约**: 用户可选择座位和时间段进行预约
- **预约取消**: 用户可取消未开始的预约
- **签到管理**: 用户到达后进行签到
- **使用结束**: 用户结束使用后释放座位
- **预约记录查询**: 查看历史预约记录

### 3.4 签到管理模块
- **签到记录**: 记录用户签到信息
- **签到详情**: 查看签到的详细信息
- **签到记录管理**: 工作人员可管理签到记录

### 3.5 系统管理模块
- **轮播图管理**: 管理首页轮播图
- **系统设置**: 管理系统基本设置
- **规则管理**: 管理预约规则
- **通知管理**: 发布和管理系统通知

### 3.6 统计分析模块
- **使用统计**: 统计座位使用情况
- **用户统计**: 统计用户活跃度

## 5. API接口设计

### 5.1 用户管理接口
- **POST /api/user/register/user**: 注册普通用户
- **POST /api/user/register/staff**: 注册工作人员
- **POST /api/user/register/admin**: 注册管理员
- **POST /api/user/login**: 用户登录
- **PUT /api/user/updatePassword**: 修改密码
- **PUT /api/user/updateInfo/user**: 更新普通用户信息
- **PUT /api/user/updateInfo/staff**: 更新工作人员信息
- **PUT /api/user/updateInfo/admin**: 更新管理员信息
- **GET /api/user/login/{loginId}**: 获取登录信息
- **GET /api/user/user/{loginId}**: 获取普通用户信息
- **GET /api/user/staff/{loginId}**: 获取工作人员信息
- **GET /api/user/admin/{loginId}**: 获取管理员信息
- **GET /api/user/canReserve/{loginId}**: 检查用户是否可以预约
- **GET /api/user/logins**: 获取所有登录用户
- **GET /api/user/logins/filter**: 根据用户名和角色筛选登录用户
- **GET /api/user/users**: 获取所有普通用户
- **GET /api/user/staffs**: 获取所有工作人员
- **GET /api/user/admins**: 获取所有管理员
- **DELETE /api/user/delete/{loginId}**: 删除用户
- **PUT /api/user/setBanStatus**: 设置用户禁止预约状态
- **GET /api/user/violation-records/{loginId}**: 获取用户的违规记录
- **GET /api/user/violation-records**: 获取所有违规记录
- **DELETE /api/user/violation-records/{id}**: 删除违规记录
- **POST /api/user/uploadAvatar**: 上传头像
- **GET /api/user/frequent-seats/{loginId}**: 获取用户的常用座位列表
- **POST /api/user/frequent-seats**: 添加常用座位
- **DELETE /api/user/frequent-seats/{id}**: 删除常用座位

### 5.2 座位管理接口
- **GET /api/seat/areas**: 获取所有区域
- **GET /api/seat/areas/{floor}**: 根据楼层获取区域
- **GET /api/seat/seats/{areaId}**: 获取区域的座位

- **POST /api/seat/reserve**: 预约座位
- **POST /api/seat/cancel**: 取消预约

- **POST /api/seat/endUse**: 结束使用
- **DELETE /api/seat/delete**: 删除预约
- **GET /api/seat/reservations/login/{loginId}**: 获取用户的预约记录
- **GET /api/seat/reservations**: 获取所有预约记录
- **GET /api/seat/area/{areaId}**: 根据区域ID获取座位（用于座位管理）
- **POST /api/seat/**: 添加座位
- **PUT /api/seat/{seatId}**: 更新座位
- **DELETE /api/seat/{seatId}**: 删除座位
- **POST /api/seat/area**: 添加区域
- **PUT /api/seat/area/{areaId}**: 更新区域
- **DELETE /api/seat/area/{areaId}**: 删除区域
- **PUT /api/seat/reservation/{reservationId}**: 更新预约状态

### 5.3 签到管理接口
- **POST /api/checkin**: 完成签到
- **GET /api/checkin/login/{loginId}**: 获取用户的签到记录
- **GET /api/checkin**: 获取所有签到记录（供工作人员使用）
- **DELETE /api/checkin/{id}**: 删除签到记录（供工作人员使用）

## 6. 核心业务流程

### 6.1 座位预约流程
1. 用户登录系统
2. 浏览可用座位
3. 选择座位和时间段
4. 提交预约请求
5. 系统验证用户是否可以预约（信用分、是否被禁）
6. 系统验证座位是否可用
7. 系统创建预约记录
8. 返回预约结果

### 6.2 签到流程
1. 用户到达图书馆
2. 打开预约记录
3. 点击签到按钮
4. 系统验证签到时间是否在允许范围内
5. 系统更新预约状态为已签到
6. 系统更新座位状态为已占用
7. 系统创建签到记录
8. 返回签到结果

### 6.3 结束使用流程
1. 用户使用完毕
2. 打开预约记录
3. 点击结束使用按钮
4. 系统更新预约状态为已结束
5. 系统更新座位状态为可用
6. 返回结束使用结果

### 6.4 违规处理流程
1. 工作人员发现用户违规行为
2. 记录违规类型和原因
3. 系统计算信用分变化和禁止天数
4. 系统更新用户信用分和禁止状态
5. 系统创建违规记录
6. 返回处理结果

## 8. 系统安全性

### 8.1 安全措施
- **密码存储**: 使用BCrypt加密存储密码，同时兼容明文密码验证（注意：生产环境应移除明文密码验证）
- **CORS配置**: 限制允许的源
- **输入验证**: 基本的请求参数验证
- **权限控制**: 基于角色的访问控制

### 8.2 潜在安全风险
- **密码验证机制**: 支持明文密码验证，存在安全隐患
- **缺少token认证**: 未使用JWT等token机制
- **缺少请求频率限制**: 可能遭受暴力攻击
- **缺少数据验证**: 可能存在SQL注入风险

## 9. 部署与运行

### 9.1 后端部署
1. 确保MySQL数据库已安装并运行
2. 创建数据库 `seat_reservation_system`
3. 执行Maven构建: `mvn clean package`
4. 运行jar包: `java -jar target/seat-reservation-system-1.0-SNAPSHOT.jar`

### 9.2 前端部署
1. 进入frontend目录
2. 安装依赖: `npm install`
3. 构建项目: `npm run build`
4. 部署dist目录到web服务器

### 9.3 运行环境
- **Java**: JDK 8+
- **MySQL**: 8.0+
- **Node.js**: 14.0+
